## How sync works now

```
Desktop enters settings → saveSettings() → KV via /dashboard/config
Phone opens dashboard → DOMContentLoaded → loadConfig() from Worker → localStorage updated
Phone opens settings → sees pre-populated fields

Desktop logs field entry → saveFieldLog() → D1 via /dashboard/data (background)
Phone opens dashboard → DOMContentLoaded → loadData() from Worker → fieldLog updated → re-render
```

**Offline fallback:** If Worker unreachable, localStorage is used. All saves go to localStorage first, then Worker async.
